Directives ========== ng-app ------ The ng-app directive is a starting point. If AngularJS framework finds ng-app directive anywhere in the HTML document then it bootstraps (initializes) itself and compiles the HTML template. ng-model -------- The ng-model directive binds HTML element to a property on the $scope object. ng-init ------- ng-init directive initializes an AngularJS Application data. It is used to put values to the variables to be used in the application. In following example, we'll initialize an array of countries. We're using JSON syntax to define array of countries.
ng-repeat --------- This directive repeats html elements for each item in a collection.

ng-bind ------- ng-bind in same way as expression {{ expression }} for example: {{Num1 * Num2}} Expression ========== Expressions are used to bind application data to html. Expressions are written inside double braces like {{ expression}}. Expressions behaves in same way as ng-bind directives. AngularJS application expressions are pure javascript expressions and outputs the data where they are used. Using numbers -------------

Expense on Books : {{cost * quantity}} Rs

Using strings -------------

Hello {{student.firstname + " " + student.lastname}}!

Using object ------------

Roll No: {{student.rollno}}

Controllers =========== ng-controller ------------- A controller is defined using ng-controller directive. AngularJS application mainly relies on controllers to control the flow of data in the application. A controller is a JavaScript object containing attributes/properties and functions. Each controller accepts $scope as a parameter which refers to the application/module that controller is to control.
